# Интерактивный каталог «Инструмент и стройка»

Интерактивный каталог по четырём типам инструмента, девяти этапам строительных работ и четырнадцати понятным пользовательским задачам.

## Запуск

Откройте `index.html` в современном браузере. Для локального сервера можно выполнить в папке сайта:

```bash
python3 -m http.server 8080
```

Затем откройте `http://localhost:8080`.

## Возможности

- переходы по типам инструмента, группам, этапам, видам работ и категориям;
- адреса в hash-маршрутах работают без серверной настройки;
- глобальный поиск с подсказками;
- фильтр операций внутри этапа;
- хлебные крошки и поддержка кнопок «назад/вперёд»;
- адаптивная версия для телефона;
- 173 нормализованные категории с прямыми переходами только на конечные `leaf`-страницы DNS;
- поисковые ссылки и ссылки на промежуточные разделы не используются;
- объединённые исходные названия разделены на самостоятельные категории;
- подбор начинается с понятной потребности пользователя, а не с названия инструмента;
- 14 готовых задач: от монтажа полки и сборки мебели до электромонтажа и мастерской;
- самостоятельный выбор одного или нескольких материалов после выбора задачи;
- выбор масштаба работ и потребности в мобильности;
- свободный выбор основных решений, альтернативные способы и полный обзор ширины подходящего ассортимента;
- кнопки перехода между шагами находятся сверху, сразу под индикатором этапов;
- финальный шаг начинается без автоматически выбранных категорий;
- демонстрационная боковая витрина открывается по нажатию на категорию;
- категории с выбранным примером товара автоматически попадают в итоговый список;
- удаление позиций, копирование и печать итогового списка;
- настройки комплектатора сохраняются в браузере;
- садовые категории отсутствуют в блоке навигации по этапам.

Исходные категории, которых нет среди конечных страниц предоставленной выгрузки DNS, не подменяются похожими разделами. Их перечень сохранён в `catalog-v2.js` в поле `meta.excludedSourceCategories`.

## Размещение на GitHub Pages

Загрузите в корень ветки `main`: `index.html`, `styles-v2.css`, `catalog-v2.js`, `app-v2.js` и `README.md`. В настройках репозитория выберите **Settings → Pages → Deploy from a branch → main / (root)**. Hash-маршруты не требуют дополнительной настройки сервера.

Дизайн использует общие паттерны крупного интернет-каталога: светлый фон, белые карточки, оранжевые акценты и сеточную навигацию. Логотипы и оригинальные изображения DNS не используются.
